# RuneScape: Dragonwilds All-in-One Server Manager

Build a complete, production-ready server manager specifically for **RuneScape: Dragonwilds dedicated servers**.

This must be a real working application, not a concept, mockup, basic launcher, collection of batch files, or generic game-server panel.

## Main objective

Create one application that allows a server owner to:

* Install the Dragonwilds dedicated server.
* Configure every server setting.
* Start, stop, restart, update, and repair the server.
* Manage worlds and backups.
* Install, update, configure, enable, disable, and remove mods.
* Connect the server to Discord.
* View live logs and server health.
* Schedule updates, restarts, announcements, and backups.
* Manage local and remote Dragonwilds servers.

The user should not need to manually edit files, install SteamCMD separately, run terminal commands, install Python, install Node.js, or set up additional utilities.

The finished program must be provided as a self-contained Windows installer or portable Windows application.

## 1. Server installation and detection

The manager must:

* Install SteamCMD automatically when needed.
* Install the RuneScape: Dragonwilds dedicated server using SteamCMD app ID `4019830`.
* Allow the user to choose the server installation folder.
* Detect an existing Dragonwilds dedicated server installation.
* Validate that all required server files exist.
* Repair or verify damaged and missing server files.
* Check for game-server updates.
* Show the installed server version.
* Show whether the server version matches the latest available version.
* Update the server through SteamCMD.
* Support automatic updates during scheduled maintenance.
* Never overwrite worlds, configurations, mods, or backups during an update.
* Display installation and update progress inside the application.

Include startup arguments such as `-log -NewConsole` through an editable launch-options page.

## 4. Visual configuration editor

Create a proper visual editor for `DedicatedServer.ini`.

Automatically locate the correct configuration path, including Windows and Linux server path variations.

The editor must support at least:

* Owner ID.
* Server name.
* Default world name.
* Admin password.
* World password.
* Maximum players.
* Server port.
* All additional settings found in the installed server version.

Requirements:

* Stop the server before applying settings that cannot safely be edited while it is running.
* Warn the user about unsaved changes.
* Validate required fields.
* Mask passwords by default.
* Provide a button to reveal or copy passwords.
* Preserve unknown configuration keys.
* Preserve settings added by future game updates.
* Create a configuration backup before every edit.
* Include a raw INI editor for advanced users.
* Include a restore-defaults option.
* Show a description and valid values for every known setting.
* Never silently delete configuration lines it does not recognize.

Do not hard-code only the current settings. Build the editor so new settings can be added through an updateable configuration schema.

## 7. Complete mod manager

Build an integrated Dragonwilds mod manager.

It must recognize and manage:

* UE4SS Lua mods.
* `.pak` mods.
* SML mods when required.
* Server-only mods.
* Client-only mods.
* Mods required by both the server and connected players.
* Mods containing configuration files.
* Mods with dependencies.

The manager must be able to:

* Install UE4SS with one click.
* Install supported mod loaders with one click.
* Import a downloaded mod ZIP.
* Detect the mod type automatically.
* Place files in the correct folder.
* Enable and disable UE4SS mods through `Mods.txt`.
* Install `.pak` files into the correct `Content/Paks/~mods` location.
* Create required mod directories.
* Update installed mods.
* Uninstall mods.
* Reinstall mods.
* Roll back a mod update.
* Back up the world before removing content-changing mods.
* Display installed version and available version.
* Show the mod author, description, requirements, dependencies, source, and last update.
* Detect duplicate files.
* Detect conflicts when multiple mods replace the same files.
* Detect missing dependencies.
* Detect mods installed in the wrong directory.
* Keep a mod installation manifest showing every installed file.
* Preserve mod configuration files during updates whenever safe.
* Allow separate mod profiles for different worlds.
* Export a mod list for players.
* Generate a player installation package or instructions for client-required mods.

Support legitimate Nexus Mods integration through an official API or authorization method where available. Do not scrape Nexus Mods, bypass download restrictions, redistribute protected files, or silently download mods against the mod author's permissions.

Where automatic downloading is unavailable, allow the user to download the archive normally and drag it into the manager for automatic installation.

Do not claim that every mod is compatible. Show compatibility as:

* Verified compatible.
* Reported compatible.
* Unknown.
* Outdated.
* Incompatible.
* Server-only.
* Client-required.

## 8. Safe mod update process

When updating a mod:

1. Stop the server if required.
2. Create a world and mod backup.
3. Read the mod manifest.
4. Remove only files belonging to the old version.
5. Install the new version.
6. Preserve compatible configuration files.
7. Validate dependencies.
8. Start the server.
9. Monitor startup logs.
10. Automatically offer rollback when startup fails.

The manager must not delete unrelated files when uninstalling or updating a mod.

* Mask server passwords.
* Use least-privilege permissions.
* Never send private server data to an outside service without permission.
* Validate ZIP extraction paths to prevent archive traversal.
* Scan imported archives for unexpected executables and suspicious paths.
* Warn before installing DLL files.
* Verify downloaded files when checksums are available.
* Maintain an audit log.
* Require confirmation for destructive actions.
* Redact secrets from exported logs and diagnostics.
* Store data locally unless the user deliberately enables external services.
